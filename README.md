# Credit Card Fraud Detection

## О проекте

В проекте проведён анализ данных и построена модель машинного обучения для выявления мошеннических транзакций.

Основная цель — оценить способность модели обнаруживать редкий класс (fraud) и изучить влияние порога классификации на качество.

---

## EDA

- Данные сильно несбалансированы (fraud ~5%)  
- Большинство признаков не имеют выраженной линейной зависимости с целевой переменной  
- Отдельные признаки (например amount) дают слабый сигнал  

---

## Модель

Использована:
- Random Forest  

Применена балансировка классов (`class_weight='balanced'`)

---

## Результаты

- Без балансировки модель игнорирует класс fraud  
- При балансировке:
  - увеличивается recall (модель находит больше мошенничества)  
  - снижается precision (много ложных срабатываний)  

- ROC-AUC ≈ 0.88 — модель хорошо различает классы на уровне вероятностей  

- Качество модели сильно зависит от выбранного порога классификации  

---

## Вывод

- Задача осложняется сильным дисбалансом классов  
- Невозможно одновременно добиться высокого precision и recall без компромисса  
- Для практического применения важно подбирать threshold в зависимости от задачи
